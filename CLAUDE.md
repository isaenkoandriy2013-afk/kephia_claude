## Project Overview

KephiaSystem is a Kerbal Space Program (KSP) planet pack mod adding a binary star system (Kephia A + Kephia B) with 23 orbital bodies. It is a pure config/asset mod ‚Äî no C# source code, no build step. The working directory is `d:\Kerbal Space Program\GameData\KephiaSystem\`.

## Testing Changes

There is no test suite or linter. To verify changes, launch KSP and observe in-game. Key mod dependencies that must be installed in `GameData/`:

- **Kopernicus** ‚Äî required for all planet configs to load
- **Scatterer** ‚Äî required for atmospheric effect configs in `Misc/Scatterer/`
- **EnvironmentalVisualEnhancements (EVE)** ‚Äî required for cloud configs in `Misc/EVE/`
- **ParallaxContinued** ‚Äî required for terrain configs in `Misc/Parallax/`

After editing `.cfg` files, delete the `Cache/` `.bin` files for affected bodies so KSP regenerates them on next load.

## Config File Format

All planet definitions use Kopernicus Module Manager patch syntax:

```cfg
@Kopernicus:NEEDS[!Kopernicus] {}   // conditional patching
@Kopernicus {
    Body {
        name = BodyName
        Properties { ... }
        Orbit { ... }
        ScaledVersion { ... }
        PQS { Mods { ... } }        // surface detail
        Atmosphere { ... }          // if atmospheric
    }
}
```

Key sections:
- **Properties**: `radius`, `geeASL`, `rotationPeriod`, `description`
- **Orbit**: `referenceBody`, `semiMajorAxis`, `eccentricity`, `inclination`, `meanAnomalyAtEpoch`
- **ScaledVersion**: Visual representation ‚Äî `type` (Star/Vacuum/Atmospheric), material colors, lens flare
- **PQS/Mods**: Surface generation ‚Äî height maps (`VertexHeightMap`), color maps (`VertexColorMap`), noise (`VertexHeightNoiseVertHeightCurve2`), biome control (`LandControl`)

## Texture Naming Convention

Textures live in `Textures/PluginData/` and follow the pattern `{BodyName}{Type}.dds`:
- `Clr` ‚Äî color/albedo map
- `Hgt` ‚Äî height map (grayscale elevation)
- `Nrm` ‚Äî normal map

Textures are referenced in configs as `BUILTIN:` or relative paths. DDS format is required for KSP; PNG is used only for special effects (flares, spikes).

## Binary Star System Architecture

Kephia A is the primary star. Kephia B orbits Kephia A. All planets orbit either Kephia A or Kephia B as their `referenceBody`. `SunFix.cfg` patches stock Sun intensity curves to balance lighting alongside the two Kephia stars ‚Äî edit this if adding new stars or adjusting luminosity.

## Cache Files

`Cache/*.bin` files are auto-generated by KSP/Kopernicus and must not be hand-edited. They should not be committed (add to `.gitignore` if not already excluded). Delete them to force regeneration after config changes.
